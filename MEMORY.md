## Key File Notes

- `scripts/extract_rgssad.py` — pure-stdlib RGSSAD v1 extractor. Run it once per Uranium release to unpack `Uranium.rgssad` → `<dir>/_unpacked/`. Algorithm: XOR stream cipher, initial key 0xDEADCAFE, advance `key = key * 7 + 3`. File data uses the global key as starting seed and advances per 4-byte group within the file; the global key is *not* advanced by file data, only by name and size fields.

## Uranium-Specific Discoveries

- **Essentials version: v17.** Evidenced by 29 sections suffixed `_v17` in `Scripts.rxdata`. See `reference/uranium_essentials_version.md`.
- **No PBS source files ship.** Only Essentials' compiled `.dat` files are distributed — see `reference/uranium_dat_inventory.md`.
- **No `pokemon.dat`/`abilities.dat` despite Essentials convention.** Species data lives somewhere else in Uranium's Data/ — candidates are `attacksRS.dat`, `tmpbs.dat`, or inside `dexdata.dat`. **Open question — needs spike before Phase 2.**
- **Map/event scale:** 199 maps, 5,301 events, 8,429 event pages. 132/199 maps have at least one page with ≥30 commands.
- **RMXP event trigger codes (got this wrong once):** `0` = Action Button (talk), `1` = Player Touch, `2` = Event Touch, `3` = Autorun, `4` = Parallel. (Map200's step-on doormat warp is trigger 1, confirming.) The talk-once NPC idiom = a 2-page event, base page trigger 0 sets self-switch A (code `123 [A,0]`), a later page gated on `condition.self_switch_valid`+`self_switch_ch`. **860** events set a self-switch via 123; **999** have a self-switch-gated page. Native global `Control Switches` (121, in-list `if`-branch) is rare (539) — the clean single-page global-switch talk-twice NPC essentially doesn't exist in Uranium.
- **Temp-switches are a SECOND, distinct self-switch-like idiom (`setTempSwitchOn`/`tsOn?`, 345× across 77 maps).** Defined in `022_Game_Event_v17.rb` (`@tempSwitches[c]=true`); `@tempSwitches` is init'd `{}` in `Game_Event#initialize` and is **NOT** in `$PokemonGlobal` (unlike `eventvars`, which IS persisted) → temp-switches are **per-map-visit runtime state that resets on map reload** = pokeemerald **TEMP flag** semantics (0x0–0x1F, auto-reset on warp), **not** persistent self-switches. Set/cleared via script call (code 355: `setTempSwitchOn("A")`/`setTempSwitchOff`), checked via `tsOn?/tsOff?` (or page conditions). **Conversion gap:** the agent maps `setTempSwitchOn("A")`→`setflag(FLAG_MAP..SSA)` (treats it as a saved self-switch — both wrong on persistence AND undefined, since orchestrator minting only covers code 123 + `self_switch_valid` pages). Disposition is a §9 #2 gate decision (TEMP-flag map vs saved self-switch vs queue). `pbSetSelfSwitch` (059/221) is the REAL persistent self-switch (→`$game_self_switches`); code 123 is its RMXP-native form (Layer 1 mints those).
- **`MapInfos` editor names are unreliable as in-game identity (Phase 5 concern).** Discovered at the Phase 3 V4 wiki spot-check: map 7's `MapInfos` name is "Comet Cave" but its content is the wiki's **Passage Cave** (BGM "PU-Passage Cave", `pbPushThisBoulder` Strength puzzle, the early cave's connection pattern). The deserializer is faithful — the names themselves are stale dev labels. Phase 5 (overworld wiring / display names) must derive location identity from content + the town-map / wiki, **not** from `MapInfos`/`map_names.json`. (`map_names.json` and `map_infos.json` agree with each other — they're the same editor source — so cross-checking them does not catch this.)
- **Scripts.rxdata has 260 sections;** ~13 are clearly Uranium-original (Nuclear_*, URANIUM_*, Bambo, Tandor, Custom_Trainers, GenderSelect, StarterSelect, etc.). See `reference/scripts_index.md` ⚠ flags. Note: `220_Custom_Mode.rb` and `222_Gym_8.rb` were in the dump but missed in the agent's `uranium_custom_features.md` summary.
- **Audio middleware: FMOD.** Uranium uses FMOD via DLL (`fmodex.dll`), wrapped by scripts 247–249. Strip — GBA cannot use FMOD.
- **EliteBattle and BW UI overrides** are third-party Essentials mods (not Uranium-original) shipped with the game; both strip on conversion since pokeemerald-expansion has its own UI/battle scene.
- **Tandor dex: 200 entries, 201 internal species IDs.** Confirmed from `dexdata.dat` (15276 bytes ÷ 76 = 201) and `regionals.dat` (200 species with Tandor numbers 1–200; 1 species has no Tandor number — identity TBD). The "~200 species, 18 Uranium-original" claim is resolved for count but fakemon identity still unconfirmed.
- **Two `.dat` binary formats.** Custom Essentials binary (`dexdata.dat`, `attacksRS.dat`, `tmpbs.dat`, `evolutions.dat`, `eggEmerald.dat`, `regionals.dat`, `moves.dat`, `items.dat`, `tutor.dat`, `tm.dat`, `metrics.dat`) — parse from `Compiler.rb` write schema, no Ruby needed. Ruby Marshal (`trainers.dat`, `encounters.dat`, `connections.dat`, `metadata.dat`, etc.) — load with stubs. See `uranium_dat_inventory.md`.

**Registry initialized (Phase 4 §4.1).** `flag_registry.py` is the sole minter. **Self-switch minting (added 2026-06-01).** `mint_self_switch(map, event, letter)` registers per-event self-switch flags (`FLAG_MAP{m:03}_EVENT{e:03}_SS{letter}`) deterministically + idempotently; the **orchestrator** derives them from each event (`_event_self_switches`: code 123 sets + self-switch page conditions) and mints on script-accept, so the **frozen prompt is unchanged**. `dump_header` now takes `flag_base`/`var_base`/`selfswitch_base` params (Phase-7 hook) and emits a `RPG2GBA_SELFSWITCH_BASE` block. **Temp-switch minting (added/committed 2026-06-03).** `mint_temp_switch(map, event, key)` mirrors `mint_self_switch` for Uranium temp-switches (`setTempSwitchOn`/`tsOn?`, script code 355/655) — per-map-visit state → pokeemerald **auto-reset TEMP flag**, named `FLAG_MAP{m:03}_EVENT{e:03}_TS{key}` (**TS** marker keeps it distinct from `SS`). Persists under `temp_switches`; `dump_header` takes a `tempswitch_base` param + emits `RPG2GBA_TEMPSWITCH_BASE` (must be the engine's auto-reset range at Phase 7).
